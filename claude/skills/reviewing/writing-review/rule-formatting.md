# フォーマットルール

記号・マークアップの使い方に関するルールです。

## 差分で検出できるルール

- 水平線（---）を視覚的な区切りとして使わない。セクションの区切りは見出しで表現する
- 太字（**）は使わない。強調が必要な箇所は文の構造や語順で伝える
- リスト項目の先頭に絵文字をラベルとして付けない（✅❌💡🔥🚀等）

## 段落の読み込みが必要なルール

- コードブロック（```）は、ソースコード・コマンド・設定ファイル・実行結果など、そのまま機械が解釈する、またはそのまま端末に貼り付ける内容にのみ使う。手順・箇条書き・説明文をインデントや等幅フォントで整形する目的では使わない。ただし、ユーザーがそのままコピーして使う出力テンプレートはコードブロックを使ってよい
- Markdownの引用記法（>）は他者の発言や外部資料からの引用にのみ使う。前提条件・注意書き・補足説明を目立たせる目的では使わない。前提条件や注意書きを区別したい場合は、見出し（##）またはGitHub Flavored Markdownのalert記法（> [!NOTE]等）を使う
- コロン（:／：）で説明やブロック要素を導くパターンは使わない。ブロック要素を導く場合は直前の文を「以下の通りです。」等で終えて段落を分ける。項目と説明の対が短い場合は表を使い、各項目に段落が必要な場合は見出しと本文で構造化する

## 全体のチェックが必要なルール

- h1からh6まで適切に使用する
