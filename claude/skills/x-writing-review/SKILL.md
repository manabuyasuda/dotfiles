`.md` ファイルの文章品質をレビュー・修正するスキルです。1周ぶんのレビューと修正は `writing-reviewer` エージェントに委譲し、このスキルは対象の決定と繰り返し制御を担います。

修正で別の問題を生むことがあるため、直接呼び出しでは3周繰り返します。各周回は独立コンテキストのエージェントが白紙から評価するので、前周の判断を引きずりません。

## 対象の決定

呼び出し元によって対象の取得元が異なります。

### format.sh 経由（PostToolUse hook）

feedbackメッセージの「ファイル: `<path>`」を対象ファイルとします。「変更差分:」が含まれる場合は、diffの`@@`ヘッダーから変更行の範囲を特定し、その前後5行をレビュー範囲とします。差分がない場合（新規ファイル等）はファイル全体を範囲とします。

### 直接呼び出し

ユーザーの指示で明示されたファイルパスまたは範囲を対象とします。引数が`staged`の場合は`git diff --cached --name-only`で対象ファイルを取得します。

## 手順

### format.sh 経由の場合

1. feedbackメッセージから対象ファイルパスとレビュー範囲を確認します。
2. `Agent`ツールで`writing-reviewer`エージェントを1回起動し、対象ファイルとレビュー範囲を渡します。
3. ステージ対象ファイルを修正した場合、`git diff -- <path>`を実行し、ステージにのみ存在する行（`-`行）が生じていないか確認します。生じていればユーザーに報告して中断します。
4. エージェントが返した修正内容の概要を報告します。

### 直接呼び出しの場合

1. `Agent`ツールで`writing-reviewer`エージェントを3回起動します。各回、対象ファイルのパス（範囲指定があれば範囲も）を渡します。

   ```
   Agent: writing-reviewer
   引数: <対象ファイルのパス>（必要なら範囲）
   ```

2. エージェントは毎回独立コンテキストで白紙から評価するため、前周の修正で生まれた新たな違反も次の周回で検出されます。
3. 3周完了後、各周回の修正内容をまとめて報告します。

## 注意

- 文章ルール（整形・構造・日本語表現）の内容は`writing-reviewer`エージェント側（`agents/writing-reviewer.md`と`x-writing-review/rule-*.md`）で管理します。
- エージェントが起動できない場合は、その旨を伝え、`writing-reviewer`の定義にしたがって手動でレビューします。
