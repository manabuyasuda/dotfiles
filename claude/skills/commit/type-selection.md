# typeの選択ガイド

コミットの変更内容にもっとも合致するtypeを選びます。commitlintの設定でtypeの集合が定義されている場合はそちらを優先してください。

| type | 用途 |
|------|------|
| `feat` | 新しい機能を追加します（fix・refactorに当てはまらない新規追加） |
| `fix` | 既存の不具合を修正します |
| `refactor` | 振る舞いを変えずにコードを整理します |
| `perf` | 処理速度・メモリなどパフォーマンスを改善します |
| `docs` | READMEやコメントなど、ドキュメントのみを変更します |
| `test` | テストコードを追加・修正します（プロダクションコードは変えません） |
| `style` | インデントや空白など、フォーマットのみを変更します（動作は変えません） |
| `build` | ビルドシステムや依存パッケージを変更します |
| `ci` | CI/CDパイプラインの設定を変更します |
| `chore` | 上記のいずれにも当てはまらない変更に使います |

## 判断に迷うケース

| 状況 | type | 理由 |
|------|------|------|
| 機能追加とバグ修正が混在 | — | コミットを分割します |
| リファクタリング中にバグを発見・修正 | `refactor` + `fix` | 目的で分けて別コミットにします |
| 新機能のテスト追加 | `feat` | 実装と同じコミットに含めます（1論理変更） |
| テストのみの追加・修正 | `test` | プロダクションコードを変えていません |
| 既存機能の改善（破壊なし） | `feat` | 新しい振る舞いの追加です |
| 設定ファイルのみの変更 | `build` / `ci` / `chore` | ビルド関連は`build`、CI関連は`ci`、その他は`chore` |
