## 主要なナビゲーションには`<nav>`を使います

ページ内のナビゲーションをランドマークとしてマークアップします。スクリーンリーダーはナビゲーションランドマークを一覧表示します。ユーザーはランドマーク間をジャンプし、目的の操作へ素早くたどり着けます。

### 対象となる要素

ページ内の主要なナビゲーションです。

- グローバルナビゲーション
- ローカルナビゲーション・サイドナビ
- パンくず（breadcrumb）
- フッター内のサイトマップ的なリンク群

### 確認するHTML要素・属性

ナビゲーションをランドマーク化していない可能性のあるパターンです。

- `<nav>`を使わず`<div>`でナビゲーションを実装している
- `<nav>`の代わりに`<div role="navigation">`を使っている
- 複数の`<nav>`を持つページで識別ラベル（`aria-label`・`aria-labelledby`）が省略されている
- ラベルが「ナビ1」「メニュー」など、内容を区別できない名称になっている

### 判定

| if | then |
|---|---|
| 主要なナビゲーションを`<nav>`で実装している場合 | 許容します |
| 主要なナビゲーションを`<nav>`以外（`<div>`・`<ul>`単独・`<div role="navigation">`）で実装している場合 | 違反です（ネイティブ要素の`<nav>`を使います） |
| 複数の`<nav>`があるページで、各`<nav>`に識別ラベルを指定している場合 | 許容します |
| 複数の`<nav>`をもつページで、識別ラベルが付いていない場合 | 違反です |
| ラベルが「ナビ1」「メニュー」など、内容を区別できない名称の場合 | 違反です |

複数の`<nav>`を使う場合は、`aria-label`で識別ラベルを付けます。各ラベルは、ユーザーが目的を区別できる名称にします。「メイン」「サブ」というUI用語よりも、「製品カテゴリー」「サポート情報」のように内容を示す名称が望ましいです。

#### Incorrect

`<nav>`を使わず`<div>`でナビを実装した例です。

```tsx
<div className="nav">
  <a href="/products">製品</a>
  <a href="/support">サポート</a>
</div>
```

#### Correct

`<nav>`を使い、複数のnavに識別ラベルを付けた例です。

```tsx
<nav aria-label="製品カテゴリー">
  <ul>
    <li><a href="/products/laptop">ノートPC</a></li>
    <li><a href="/products/desktop">デスクトップPC</a></li>
  </ul>
</nav>
<nav aria-label="サポート情報">
  <ul>
    <li><a href="/support/faq">よくある質問</a></li>
    <li><a href="/support/contact">お問い合わせ</a></li>
  </ul>
</nav>
```

実装後はブラウザのアクセシビリティパネルでランドマーク一覧を表示し、各navが識別可能な名前になっているか確認します。
