## ページ遷移には`<a href>`を使います

ユーザーが別URL（外部URL・別ページ・同ページ内アンカー）へ移動する手段になる要素です。新規タブで開く・URL共有・ブラウザバックといったリンクの基本機能は`<a href>`が提供します。

### 対象となる操作

ユーザーをある場所から別の場所へ「移動」させる操作です。URLが変わるかどうかは判定基準ではありません。

- 外部URL（別オリジン）への遷移
- 別ページ（同オリジンの別パス）への遷移
- 同ページ内の特定位置へのスクロール（「現時刻へ」「ページトップへ」など）

>[!NOTE]
>同ページ内スクロールは`scrollTo()`で実装するとURLハッシュが変わらない場合があります。しかしURLが変わらなくても「移動先がある」操作であれば`<a href="#id">`が正しい実装です。タグの選択は実装手段ではなく操作の意味で決めます。

### 確認するHTML要素

- `<button onClick>`から`router.push`・`location.href`・`window.open`等を呼んでいる
- `<div onClick>`・`<span onClick>`で同様の処理をしている
- 同じ要素内に複数のリンク先がある（カード内では詳細・著者・タグなど）

### 判定

| if | then |
|---|---|
| クリックでURLが変わり、`<a href>`を使っている場合 | 許容します |
| URLが変わるクリック操作を`<button>`や`<div>`で実装している場合 | 違反です |
| 同じ要素内に複数のリンク先があり、主要素のみ`<a>`で囲んでいる場合 | 許容します |
| `<a>`をネストして実装している場合 | 違反です |

`<a>`のネストはHTML仕様違反です。リンク先が複数ある場合は、カード全体のクリックを擬似要素で実現します。子要素を`<a>`の入れ子にしないでください。

#### Incorrect

URL遷移を`<button>`で実装した例です。新規タブで開く・URL共有・ブラウザバックが機能しません。

```tsx
<button onClick={() => router.push('/news/123')}>ニュース詳細</button>
```

#### Correct

`<a href>`で実装した例です。

```tsx
<a href="/news/123">ニュース詳細</a>
```

実装後はEnterキー押下・ブラウザバック・新規タブ表示の各動作が想定通り機能するか確認します。
