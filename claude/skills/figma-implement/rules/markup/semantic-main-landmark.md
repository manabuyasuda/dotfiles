## ページのメインコンテンツは`<main>`で囲みます

ページ固有の主要コンテンツを示すランドマーク要素です。`<main>`で囲むとUAやスクリーンリーダーが「メインコンテンツへスキップ」のような機能を提供できます。

### 対象となる範囲

ページ固有の主要コンテンツです。各ページで異なる中核部分を指します。

- 記事本文・商品詳細・ユーザープロフィール
- 一覧ページの本体（記事一覧・検索結果・商品一覧など、リスト・絞り込み・ページネーションを含む）
- ダッシュボードのメインパネル

次の要素は`<main>`の外に出します。

- グローバルナビゲーション（`<header>`内や独立した`<nav>`に配置）
- サイドバー（`<aside>`に配置）
- フッター・著作権表記（`<footer>`に配置）
- パンくず（独立した`<nav aria-label="パンくず">`に配置）

ただしデザイン上で外に出せないパンくずは、`<main>`内に置くことも許容します。

### 確認するHTML要素

`<main>`が正しく使われていない可能性のあるパターンです。

- ページに`<main>`がない（`<div id="main">`等で代用している）
- ページに`<main>`が複数ある
- `<main>`の中にサイト共通要素（ヘッダー・サイドバー・フッター）を含めている
- `<section>`や`<article>`の代わりに`<main>`を使っている

### 判定

| if | then |
|---|---|
| ページの主要コンテンツを`<main>`で1つだけ囲んでいる場合 | 許容します |
| `<main>`がなく`<div>`で代用している場合 | 違反です |
| `<main>`が1ページに複数ある場合 | 違反です |
| `<main>`の中にサイト共通要素（ヘッダー・サイドバー・フッター）が含まれている場合 | 違反です |
| セクション区切りや記事ブロックに`<main>`を流用している場合 | 違反です |

`<main>`はページの中核を示すランドマークです。`<section>`や`<article>`の代わりに使うものではありません。

#### Incorrect

サイト共通のロゴとグローバルナビを`<main>`内に含めた例です。

```tsx
<main>
  <a href="/"><img src="/logo.svg" alt="サイト名" /></a>
  <nav aria-label="グローバル">
    <ul>
      <li><a href="/about">会社情報</a></li>
    </ul>
  </nav>
  <article>本文</article>
</main>
```

#### Correct

サイト共通要素を`<main>`の外に出した例です。

```tsx
<header>
  <a href="/"><img src="/logo.svg" alt="サイト名" /></a>
  <nav aria-label="グローバル">
    <ul>
      <li><a href="/about">会社情報</a></li>
    </ul>
  </nav>
</header>
<main>
  <article>本文</article>
</main>
<footer>...</footer>
```

実装後はブラウザのアクセシビリティパネルで`<main>`を確認します。1ページに1つあり、サイト共通要素を含んでいないことを確認します。
