## 画像のalt属性は役割と文脈で記述します

alt属性は画像の意味を伝えます。スクリーンリーダーや画像が表示されない環境のユーザーへ届きます。役割と文脈に合わせて記述します。SEO目的のキーワードや、画像が持たない情報を盛り込みません。

### 対象となる画像

すべての`<img>`要素です。`alt`属性は必須で、省略するとスクリーンリーダーがファイル名を読み上げます。

### 確認するHTML要素・属性

altが正しく書かれていない可能性のあるパターンです。

- `alt`属性が省略されている
- `alt="画像"`・`alt="写真"`・`alt="image.png"`のような汎用名・ファイル名になっている
- 装飾画像に説明テキストが入っている
- SEO目的のキーワードが詰め込まれている
- 商品一覧などで前後の文章と重複した内容がaltに入っている
- グラフ・チャートで詳細データへの言及がない
- キャプション（`<figcaption>`等）と重複した内容がaltに入っている

### 判定

| if | then |
|---|---|
| 意味のある画像に内容を簡潔に説明するaltがある場合 | 許容します |
| 装飾画像に`alt=""`が指定されている場合 | 許容します |
| `alt`属性が省略されている場合 | 違反です |
| altが「画像」「写真」のような種別のみ、またはファイル名の場合 | 違反です |
| altにSEO目的のキーワードが盛り込まれている場合 | 違反です |
| 装飾画像に説明テキストが入っている場合 | 違反です |
| キャプション（`<figcaption>`等）と同じ内容をaltに書いている場合 | 違反です（altは`alt=""`にして、説明はキャプションに任せます） |
| グラフのデータがテキスト情報として別途あり、altにそのデータ参照先を示している場合 | 許容します |
| グラフのaltが概要・主要データ・補足の順で説明している場合 | 許容します |

### 書き方

- 「画像」と書かず、「写真」「グラフ」「イラスト」「〜の様子」のような具体的な言葉を使います
- 種別と内容を組み合わせて記述します。「写真：チームの集合」「グラフ：売上推移」のような形です
- テキストの画像化は、altに同じテキストを使います
- 自然な読み上げのためにテキストを変更しません。スクリーンリーダーや読み手による変換に任せます
- 文章を補足する画像のaltは、前後の文章と続けて読んで自然になるよう選びます
- クリッカブルマップ内の`<img>`は画像全体を説明します。各`<area>`は対応するリンク先を説明します

#### Incorrect

altが正しく書かれていない例です。

```tsx
<img src="/team.jpg" />
<img src="/team.jpg" alt="team.jpg" />
<img src="/team.jpg" alt="画像" />
```

装飾画像に説明テキストが入っている例です。

```tsx
<img src="/divider.svg" alt="区切り線" />
```

#### Correct

意味のある画像に種別+内容で記述した例です。

```tsx
<img src="/team.jpg" alt="写真：5人のチームメンバーが会議室で議論する様子" />
```

装飾画像に空のaltを指定した例です。

```tsx
<img src="/divider.svg" alt="" />
```

グラフ画像で、データが本文中にテキストとして存在する例です。

```tsx
<img
  src="/sales-chart.png"
  alt="グラフ：四半期別売上の推移（データは本文の表を参照）"
/>
```

実装後はスクリーンリーダー（macOS: VoiceOver、Windows: NVDA）で画像箇所の読み上げ内容を確認します。ブラウザの設定で画像をオフにしてaltテキストが表示されるかも確認します。axeまたは`eslint-plugin-jsx-a11y`の`alt-text`ルールで静的検出もできます。
