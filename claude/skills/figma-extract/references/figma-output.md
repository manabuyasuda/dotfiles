# Figma出力の読み方

## sparseレスポンスの処理

`get_design_context`はノードが大きい場合、完全なJSXコードではなくsparse（座標・子ID一覧のみのXML）を返します。

| 返り値の種類 | 判定方法 |
|---|---|
| 完全レスポンス | `export default function`を含むJSXコード |
| sparse | XML属性リスト・座標情報のみ |

取得を開始する前に`_index.json`を確認します。記録済みのノードは再取得する必要がありません。

sparseを受け取ったら、以下の手順で子ノードを分割して取得します。

1. ルートノードを取得します。sparseなら子node-idを`_index.json`に記録し、`rawFile`は`{nodeId}.xml`として保存します
2. sparse XMLに列挙されているすべての子ノードIDを`pendingNodes`に追加します
   - 用途・重要度・実装する予定かどうかで選別しません。全件を追加します
   - `userSkippedNodes`に登録済みのノードIDだけは追加を省略します
   - 追加件数とsparse XMLの子ノード数の差が`userSkippedNodes`の重複件数と一致することを確認してから次に進みます

### 同一コンポーネントの繰り返し行（代表フェッチ）

sparse XMLの子ノードに、同一コンポーネント名の行が複数並ぶことがあります（タイムライン行・競技別行など）。この場合は以下の手順を取ります。

1. 同一名の行が複数あることをユーザーに示し、「代表1件のみ取得して残りをskipしてよいか」を確認します
2. ユーザーが承認した場合は代表1件を取得し、残りを`userSkippedNodes`に記録します
3. ユーザーが承認しなかった場合は全件を`pendingNodes`に追加して取得します

ユーザー確認を省略して独自判断でスキップしないでください。繰り返し行の存在を発見した時点で必ず確認します。
3. 各子ノードを`get_design_context`で取得します
4. 完全レスポンスが返れば`{nodeId}.txt`に保存して`_index.json`を更新します
   - Pass 1: JSX内のすべての`data-node-id`をgrepして`jsxNodes`に記録します（判断なし・全件）
5. sparseが返れば子node-idをすべて記録してさらに取得します（再帰）
6. `pendingNodes`が空になれば取得が完了します
7. Pass 2: `jsxNodes`のすべてのエントリを参照してコンポーネント境界を判断し`componentNodes`に記録します
   - Pass 2は`pendingNodes`が空になってから1回だけ実行します

## テキストの改行

Figmaのテキストノードに`\n`（改行）が含まれると、`get_design_context`の出力では複数の`<p>`タグに展開されます。Figmaで「テキストとしてコピー」すると改行が消えて1行に見えますが、JSX上では複数行になります。テキスト表には展開後の各`<p>`を別行として記録します。

```
// Figmaから文字列コピー
"ラベルA サブラベルB"  ← 改行が消えて1行に見える

// get_design_context の出力
<p>ラベルA</p>
<p>サブラベルB</p>  ← 2行に展開されている
```

| ノード名 | テキスト内容 | 表示行数 |
|---|---|---|
| headerCell | "ラベルA" / "サブラベルB" | 2行 |

## コンテナーとテキストノードの区別

タイポグラフィは`data-node-id`で特定したテキストノード（`<p>` / `<span>`）自身のスタイルから読みます。Figmaの出力では親コンテナーと子テキストは別ノードになっており、コンテナーに`leading-[0]`が含まれることもあります。これはレイアウト上の配置トリックであり、テキストのlineHeightではありません。

```
親コンテナー: leading-[0]    ← 配置トリック（タイポグラフィとして読まない）
  子テキスト: leading-none  ← こちらが実際のlineHeight
```

## レイアウトの算出

対象ノードの座標（x, y, width, height）と親ノードのサイズを照合して余白を算出します。

- `get_design_context`がsparseを返した場合は、そのXML内の座標情報を使います
- 完全レスポンスが返った場合、親ノードのサイズが含まれていないことがあります。その場合は`get_metadata`で親ノードを取得してください

```
親width=375、子x=16 width=343
  → 左余白=16、右余白=375-343-16=16 → paddingInline: 16px（左右均等）

親width=375、子x=0 width=291
  → 左余白=0 → 左寄せ、または親に別の要素が並んでいる

親width=375、子x=329 width=46
  → 左余白=329 → 右寄せ（marginInlineStart: auto など）

親height=82、子y=16 height=50
  → 上余白=16、下余白=82-50-16=16 → paddingBlock: 16px（上下均等）
```

算出結果はマッピングファイルのスタイル表に記録します。

| プロパティ | Figma値 | トークン | 状態 |
|---|---|---|---|
| layout | 親375px中 x=16 w=343 → paddingInline 16px | | |
