計画・設計の失敗シナリオを着手前に洗い出すスキルです。分析の実体は `pre-mortem` エージェントに委譲し、このスキルは対象の特定と結果の提示を担います。

エージェントに切り出しているのは、失敗シナリオの列挙が対話を伴わないread-onlyの分析で、計画系スキルからも部品として再利用できるためです。

## 手順

1. 引数から対象の計画・設計ファイルのパスを読み取ります。渡されていない場合は、`Glob`で計画・設計ファイル（`plan/`・`docs/`・`*.md` など）を探します。候補が複数あって絞れないときは、ユーザーに対象を確認します。
2. `Agent`ツールで`pre-mortem`エージェントを起動し、対象ファイルのパスを渡します。

   ```
   Agent: pre-mortem
   引数: <対象の計画・設計ファイルのパス>
   ```

3. エージェントが返した失敗シナリオ（技術的・運用的・認知的）と着手前に決めておくべき対策を、そのままユーザーに提示します。

## 注意

- このスキルでは分析ロジックを持ちません。失敗シナリオの分類・書式・判断基準は`pre-mortem`エージェント側で管理します。
- エージェントが起動できない場合は、その旨を報告し、`pre-mortem`エージェントの定義（`agents/pre-mortem.md`）にしたがって手動で分析します。
