大きなタスクを最小単位のステップに分解するスキルです。分解の実体は`decompose`エージェントに委譲し、このスキルは対象の特定と結果の提示・タスク登録を担います。

エージェントに切り出しているのは、分解が対話を伴わないread-onlyの分析で、計画系の処理から部品として再利用でき、Claudeが「タスクが大きい」と判断したときに自律的に呼べるためです。

## 手順

1. 引数から対象の計画・実装タスクを特定します。ファイルがある場合は`Glob`で探します。
2. `Agent`ツールで`decompose`エージェントを起動し、対象タスクを渡します。

   ```
   Agent: decompose
   引数: <対象の計画・タスク、またはそのファイルパス>
   ```

3. エージェントが返したステップ（依存順・並列可否つき）を提示します。続けて着手する場合は、各ステップを`TaskCreate`で登録します（順序の依存は`blockedBy`で表現します）。

## 注意

- 分解の基準・出力書式は`decompose`エージェント側で管理します。
- エージェントが起動できない場合は、その旨を伝え、`decompose`エージェントの定義（`agents/decompose.md`）にしたがって手動で分解します。
