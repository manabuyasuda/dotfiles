# 日本語表現ルール

日本語の表現・語彙の使い方に関するルールです。

## 差分で検出できるルール

- 英数字・記号と日本語の間に半角スペースを入れない。バッククォートで囲まれたインラインコードの前後も同様。
  - 正: 「Claude Codeを使う」「`true`の場合」「`config.json`を読み込む」「`true`です」
  - 誤: 「Claude Code を使う」「`true` の場合」「`config.json` を読み込む」「`true` です」
- 冗長な表現は短くする（「することができます」→「できます」、「まず最初に」→「まず」、「する必要があります」→「してください」）
- 文は述語（動詞・形容詞・助動詞）で終える。名詞で文を終えて述語を省略しない。述語がないと時制（する／した）や意図（する／できる／したい）が曖昧になる
- 漢語の複合名詞を動詞句で表現する。「未〜」「〜外」「〜値」のような漢語複合名詞は、動詞句に置き換えて自然な日本語にする（「未取得」→「取得できていない」、「定義外」→「対応していない」、「未知値」→「対応していない値」）

## 段落の読み込みが必要なルール

- 名詞を別の名詞に直接かける形（複合名詞）は使わず、助詞（の・による・に関する等）または活用形（した・する・な等）を補って修飾関係を明示する
- 「場合」と「とき」を使い分ける。条件・仮定的な状況には「場合」を使い、時間的な局面・瞬間には「とき」を使う（「残高が不足している場合は」「エラーが発生したときは」）
- テスト名など技術的な文脈では、技術的な語（`null`・`true`・`false`・数値コード等）をドメイン語で表現し、技術語は括弧内に補足する（「null を返す」ではなく「ラベルを表示しない（null）」）
- 実装の内部事情（「当日の日付を固定するため」「フラグがセットされているため」等）を説明文に含めない。説明すべきは振る舞いであり、理由を知りたい人は実装を読む
