## 画像の解像度はsrcsetとsizesで最適化します

`srcset`と`sizes`は表示領域に合わせた解像度の画像を配信する属性です。デバイスやビューポートのサイズに応じて、過大な画像を読み込まないよう最適化します。

### 対象となる画像

可変サイズの画像と、解像度を最適化したい画像です。

- ヒーロー画像・本文画像・サムネイル
- レスポンシブレイアウトで横幅が変わる画像
- 表示サイズが固定でも、高解像度ディスプレイ用に複数解像度を用意したい画像（`<img>`タグのロゴ・アイコンなど）

次の画像は対象外です。

- SVG（ベクター形式で解像度に依存しない）
- CSSの`background-image`で配置している画像（`<img>`要素ではない）

### 確認するHTML要素・属性

最適化が不十分な可能性のあるパターンです。

- `<img>`に`src`のみで`srcset`が指定されていない
- `srcset`の`w`記述子を使い、`sizes`が省略されている（デフォルト`100vw`になる）
- `sizes`の指定値と実際の表示領域が一致しない（左右余白を考慮していない等）
- 表示領域が大きく異なる画像で`x`記述子のみを使っている

### `w`記述子と`x`記述子の使い分け

表示領域がデバイスやビューポートで大きく変わるかどうかを基準に選びます。

| 表示領域の特性 | 使う記述子 | 例 |
|---|---|---|
| デバイスやビューポートで大きく変わる | `w`記述子と`sizes`を併用 | レスポンシブな本文画像・ヒーロー画像 |
| 固定または変動が小さい | `x`記述子 | 固定サイズのロゴ・アイコン |

`x`記述子は必要な画像数が最小限で済みます。表示領域の差が大きい場合は適していません。いずれかのデバイスで密度が過不足になります。

### 判定

| if | then |
|---|---|
| 表示サイズが固定または変動の小さい画像で、`srcset`に`x`記述子を指定している場合 | 許容します |
| 表示サイズが大きく異なる画像で、`srcset`に`w`記述子と`sizes`を指定している場合 | 許容します |
| 表示サイズが大きく異なる画像で、`x`記述子のみを使っている場合 | 違反です（表示領域サイズに対する最適化ができません） |
| `w`記述子を使うのに`sizes`が省略されている場合 | 違反です（`sizes`のデフォルトが`100vw`になり、過大な画像が読み込まれます） |
| `sizes`が実際の表示領域と一致しない場合 | 違反です（過大または過小な画像が読み込まれます） |

画像コンポーネントを使う前に、プロジェクトの既存ラッパー（独自の`<Image>`等）がないか確認します。既存があればそれを使います。なければ、新規作成する前にユーザーへ確認します。Next.jsの`<Image>`は`srcset`を自動生成します。`deviceSizes`と`imageSizes`をプロジェクトに合わせて調整します。

#### Incorrect

`srcset`の`w`記述子を使っているのに`sizes`が省略されている例です。デフォルト`100vw`になり、左右余白を考慮しない大きな画像が読み込まれます。

```html
<img
  src="image_320w.webp"
  srcset="image_320w.webp 320w, image_768w.webp 768w, image_1024w.webp 1024w"
  alt=""
/>
```

表示サイズが大きく変わる画像に`x`記述子だけを指定した例です。表示領域サイズに合わせた最適化ができません。

```html
<img src="hero.webp" srcset="hero.webp 1x, hero@2x.webp 2x" alt="ヒーロー画像" />
```

#### Correct

`sizes`で実際の表示領域を指定した例です（可変サイズ画像）。

```html
<img
  src="image_320w.webp"
  srcset="image_320w.webp 320w, image_768w.webp 768w, image_1024w.webp 1024w"
  sizes="(max-width: 767px) calc(100vw - 40px), 728px"
  alt=""
/>
```

固定サイズで`x`記述子のみを指定した例です（ロゴ・アイコンなど）。

```html
<img src="/logo_80w.png" srcset="/logo_80w.png 1x, /logo_160w.png 2x" alt="サービス名" width="80" height="40" />
```

実装後は開発者ツールのNetworkタブで、ダウンロードされた画像が想定通りか確認します。デバイスピクセル比とビューポート幅を変えながら検証します。
