## 見出しはh1から順番に使います

ページとコンポーネントの構造をユーザーに伝える要素です。スクリーンリーダーは見出しレベルでアウトラインを構築します。ユーザーは見出しジャンプで読みたい箇所へ移動します。

### 対象となる要素

ページとコンポーネントの全見出しです。テキストとして「見出し」と認識される箇所が対象になります。

- ページタイトル（最上位の見出し）
- セクションタイトル・コンポーネントタイトル
- サブセクション内の見出し
- カードのタイトル

### 見出しの判定基準

次のいずれかに該当する文字列を見出しとして実装します。

- デザインカンプで見出し用のテキストスタイル（Heading・H1など）が割り当てられている
- 後続コンテンツに名前を付ける、または要約する役割の文字列
- 目次やアウトラインに含まれることが想定される
- ジャンプリンク・スクロールスパイなどの遷移先になる
- スクリーンリーダーの「見出しジャンプ」で移動先として期待される

見出しと強調の判定が難しい場合は、ユーザーに確認します。単に視覚的に大きい・太いだけの装飾テキストは見出しにしません。

### 確認するHTML要素

見出しが正しく実装されていない可能性のあるパターンです。

- 見出しを`<div>`や`<p>`で実装している（フォントサイズだけ大きくしている）
- ページにh1がない、または複数ある
- h1の次がh3になっている（h2をスキップ）
- コンポーネント単体で見出しレベルを決め打ちしている（ページ全体の階層と整合しない）

### 判定

| if | then |
|---|---|
| ページのh1は1つで、見出しがh1から順番に使われている場合 | 許容します |
| ページにh1がない場合 | 違反です |
| ページにh1が複数ある場合 | 違反です |
| 見出しレベルをスキップしている（h1の次にh3など）場合 | 違反です |
| 見出しを`<div>`・`<p>`・`<span>`で実装している場合 | 違反です |

コンポーネント単位で見出しを書く場合は、配置先ページの階層に合わせます。決め打ちにせず、propsでレベルを受け取って実装します。

#### Incorrect

見出しを`<div>`で実装し、見出しレベルが文書構造に反映されない例です。

```tsx
<div className="title">記事タイトル</div>
<div className="subtitle">概要</div>
```

#### Correct

`<h1>`から順番に見出しタグを使った例です。

```tsx
<h1>記事タイトル</h1>
<h2>概要</h2>
<h2>本文</h2>
<h3>本文の小見出し</h3>
```

実装後はブラウザのアクセシビリティパネルまたはアウトラインビューで見出し階層を確認します。h1が1つだけあり、各レベルの順番を確認します。
