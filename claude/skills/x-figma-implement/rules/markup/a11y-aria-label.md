## 可視テキストがある要素にaria-labelを付けません

`aria-label`はアクセシブル名を上書きします。子孫に可視テキストがある要素に付けると、見えているテキストと読み上げられるテキストが乖離し、スクリーンリーダーユーザーが混乱します。

### 対象となる要素

`aria-label`を持つすべての要素です。

- 子孫にテキストコンテンツを持つ`<a>`・`<button>`
- `alt`属性に意味のあるテキストが入った`<img>`を子孫に持つ要素
- `<input>`のように関連する`<label>`要素が正しく設定されている要素

### 判定

| if | then |
|---|---|
| 要素に可視テキストの子孫がなく、アクセシブル名が伝わらない場合 | `aria-label`を付けます |
| 子孫に可視テキストがあり、その内容でアクセシブル名が十分伝わる場合 | `aria-label`は不要です（付けると違反です） |
| 子孫の可視テキストだけでは文脈が不足する場合（「もっと見る」ボタンが複数あるなど） | `aria-labelledby`で見出しと関連付けるか、`aria-label`で補完します |
| アイコンのみで可視テキストがない場合 | `aria-label`を付けます |

#### Incorrect

子孫に可視テキストがあるのに`aria-label`で上書きしている例です。

```tsx
<a href="/products/123" aria-label="商品名 ¥1,000">
  <span>商品名</span>
  <span>¥1,000</span>
</a>
```

#### Correct

可視テキストをそのままアクセシブル名として使う例です。

```tsx
<a href="/products/123">
  <span>商品名</span>
  <span>¥1,000</span>
</a>
```

アイコンのみで可視テキストがないボタンに`aria-label`を付ける例です。

```tsx
<button type="button" aria-label="検索">
  <img src="/search.svg" alt="" aria-hidden="true" />
</button>
```

実装後はブラウザのアクセシビリティパネルで要素のアクセシブル名を確認します。可視テキストと読み上げ内容が一致していることを確認します。
