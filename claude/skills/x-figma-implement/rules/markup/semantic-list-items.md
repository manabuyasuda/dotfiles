## 同形項目の繰り返しには`<ul>`/`<ol>` + `<li>`を使います

同じ形式の項目が複数並んでいることをユーザーに伝える要素です。スクリーンリーダーが「リスト」「項目1/N」のように構造をアナウンスします。

### 対象となる構造

同じ形式の項目が2件以上並んでいる箇所です。

- お知らせ一覧・記事一覧・検索結果
- カードグリッド・サムネイル一覧
- ナビゲーション項目（グローバルナビ・サブナビ・パンくずなど）
- タグ一覧・カテゴリー一覧

### 確認するHTML要素

`<ul>`・`<ol>`なしで項目を並べているパターンです。

- `<div>`の連なりでリスト風に表現している
- ナビゲーション項目を`<ul>`なしで`<a>`の連続にしている
- 項目自体がクリックで遷移するのに、`<a href>`を使っていない

### 判定

| if | then |
|---|---|
| 同形項目を`<ul>`または`<ol>` + `<li>`で実装している場合 | 許容します |
| 同形項目を`<div>`の並びで実装している場合 | 違反です |
| 順序に意味がある並び（ランキング・優先度順など）を`<ul>`で実装している場合 | 違反です（`<ol>`にします） |
| 各項目が遷移可能なのに`<a href>`を使っていない場合 | 違反です（`semantic-link-href`を参照） |

#### Incorrect

`<div>`を並べて見た目だけ再現した例です。スクリーンリーダーは「リスト」「項目数」をアナウンスしません。

```tsx
<div className="news-list">
  {news.map((item) => (
    <div key={item.id} className="news-item">
      <span>{item.title}</span>
    </div>
  ))}
</div>
```

#### Correct

`<ul>`と`<li>`で構造化し、各項目を`<a>`で遷移可能にした例です。

```tsx
<ul className="news-list">
  {news.map((item) => (
    <li key={item.id}>
      <a href={`/news/${item.id}`}>{item.title}</a>
    </li>
  ))}
</ul>
```

実装後はブラウザのアクセシビリティパネルで「list」ロールと項目数が正しくアナウンスされるか確認します。
