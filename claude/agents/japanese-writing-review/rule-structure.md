# 文書構造ルール

文書の構造・論点の組み立てに関するルールです。このファイル内の例示はルール定義のための禁止表現サンプルなので、該当するtextlintルールはファイル先頭で無効化しています。

## 差分で検出できるルール

- 誇張表現・ハイプ的な表現は使わない（「革命的な」「ゲームチェンジャー」「究極の」「完全に解決」「パラダイムシフト」等）
- 評価を上乗せするだけで実質の情報を加えない表現は削除し、具体的な事実で代替する（「〜の証として」「〜において重要な役割を果たしている」「〜を象徴している」「〜を体現している」等）
- 根拠のない楽観的な結語は削除する（「今後の発展が期待されます」「さらなる可能性を秘めています」「引き続き注目が集まります」等）
- 「本質的には」「真に重要なのは」「核心にあるのは」「突き詰めると」のような深みを演出するだけのフレーズは削除する。続く内容はたいていそのまま書ける
- 調査や執筆の経緯（「以前調べたときは」「当初はXと考えましたが」「再調査した結果」等のパターン）は書かない。現時点で正しい結論だけを書く。仕様や事実が変わったことに意味がある場合は客観的な事実の変化として書く（「v2.0でAからBに変更された」等）
- これから何をするかを宣言するだけの前置き文は削除し、内容に直接入る（「それでは〜について説明します」「まず〜について見ていきましょう」「以下に〜を示します」等）
- 「専門家によると」「業界関係者によれば」「有識者は〜と指摘している」のような出典のない言及は、著者・機関・文書・日付を明記するか削除する
- 文末はです・ます調で記述する。だ・である調（「〜だ」「〜である」「〜した」「〜する」「〜している」など）が含まれる場合は、無条件にです・ます調（「〜です」「〜ます」「〜しました」「〜します」「〜しています」など）に修正する

## 段落の読み込みが必要なルール

- 主語が不明な受動態は避け、動作の主体を明示する（「処理が行われます」→「システムが処理します」）
- 抽象的な形容に頼らず、定量的・具体的に書く（「高速な」→「50ms未満の」、「大幅に向上」→「従来比200%向上」）
- 1つの段落では、1つの論点とその根拠や具体例だけを扱う。論点が変わったら段落を分け、段落の間は1行空ける。検証手順として、各段落の先頭文だけを並べたときに要約として筋が通るかを確認する。通らない場合は段落の分け方か先頭文を直す
- 「なお、」「また、」「さらに、」「加えて、」が複数の文の文頭に連続して使われている場合、後続の文を文頭接続詞なしで書き直すか、前の文に統合する
- 見出しの直後の文が見出しをそのまま言い換えているだけの場合は削除する（「〇〇は重要な概念です」「〇〇について説明します」等）

## 全体のチェックが必要なルール

- 同一の対象には同一の用語を使う（文書内で「ユーザー」と「クライアント」のように表記を揺らさない）
- ドキュメントには信頼性のある情報のリンクと引用を積極的に記載する

  引用フォーマット:

  ```
  ＜英語の引用＞
  ＜日本語訳（日本語を引用する場合は省略）＞
  ＜[引用した情報のタイトル](URL)＞
  ```
